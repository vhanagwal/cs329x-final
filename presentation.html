Introduction

The Problem: Chat as Cognitive Bottleneck

Current State

  • LLM interfaces rely on linear, text-based chat
  • Users must maintain mental models of goals, revisions, and task states
  • No spatial structure despite cognition thriving on it
  • Cognitive offloading is underutilized
Key Insight: Chat creates an invisible bottleneck‚Äîusers think with the interface, not through it.

Our Approach: GenUI

  • LLM generates structured workspace layouts
  • Interfaces personalized to cognitive style
  • Externalizes reasoning as panels, boards, timelines
Research Question
Can generative, personalized interfaces enable users to complete cognitively complex tasks more effectively than chat?

Motivation grounded in distributed cognition (Risko & Gilbert, 2016) and cognitive load theory (Sweller, 1988)

Discussion

Discussion, Limitations & Future Work

Key Takeaways

  • GenUI outperforms chat on all 5 evaluation factors
  • Personalization matters: +162% on persona fit, +29% overall vs generic
  • GPT-4 can act as interface designer given structured prompts + RAG context
  • LLM-as-Judge enables scalable evaluation

Limitations

  • Synthetic users: Personas are simulated; real behavior is richer/noisier
  • LLM judge bias: May compress score ranges, inherit model biases
  • Single-run N=1: Needs N‚â•30 for statistical power
  • Domain specific: Only tested on writing tasks

Future Work

1. User Studies
IRB-approved study with real users to validate LLM judge ratings against human preferences
2. Longitudinal Adaptation
Learn from interaction history over time; co-adaptive human-AI dynamics
3. Domain Expansion
Test on design ideation, research synthesis, code editing
4. Accessibility Audit
WCAG 2.2 compliance verification for generated layouts
Vision: Generative interfaces as a new design grammar for AI‚Äîturning invisible thought processes into visible, manipulable structures.
